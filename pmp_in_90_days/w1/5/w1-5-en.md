## üéØ Critical Point: What Does the Exam Assume?

### ‚ö†Ô∏è Very Important Question:

**"If the question doesn't specify the organizational structure type, what should you assume?"**

### The Answer:

When the exam asks:
- "You are a Project Manager..."
- **Without specifying** the organizational structure type

**Always assume**: You work in **Strong Matrix**

---

## üí° Why Strong Matrix?

### Explanation:

In **Strong Matrix**:
- ‚úÖ Project Manager has **high authority**
- ‚úÖ Functional Manager **exists** but with limited authority
- ‚úÖ This is the **default assumption** in **most exam questions**
- ‚úÖ Unless the question **explicitly states** another structure type

### Example:

**If the question says**:
"You are a Project Manager working in a company with a Weak Matrix structure..."

‚Üí Here it **specifies** the structure, so answer based on **Weak Matrix**

**If the question says**:
"You are a Project Manager at company X..."

‚Üí Doesn't specify the structure, assume **Strong Matrix**

## üî§ Key Terms

### 1. Project Manager

**Abbreviation**: PM
**Role**: Manage project from start to finish
**Authority**: Varies by organizational structure

---

### 2. Project Coordinator

**Abbreviation**: None
**Role**: **Coordination only** (no real authority)
**Where Found**: Functional or Weak Matrix
**Note**: **Less than** Project Manager

---

### 3. Project Expeditor

**Definition**: Person who helps **expedite** task completion
**Role**: Follow-up and acceleration only (no executive authority)
**Where Found**: Functional or Weak Matrix
**Note**: **Less than** Project Coordinator

**Hierarchy**:
```
Project Expeditor < Project Coordinator < Project Manager
```

---

### 4. Functional Manager

**Definition**: Department manager (HR Manager, IT Manager, Finance Manager...)
**Role**: Manage a specific functional department
**Relationship with Projects**: Varies by organizational structure

**In Functional**: **All authority**
**In Weak Matrix**: **Most authority**
**In Balanced Matrix**: **Shared authority**
**In Strong Matrix**: **Advisory role**
**In Projectized**: **Not present** (or minor administrative role)

---

### 5. Package Manager

‚ö†Ô∏è **Important Warning**:

**Package Manager** = **NOT related to project management!**

- This is a term from software development
- Example: npm (Node Package Manager)
- **May appear as wrong answer** in exam as distractor

## üéì Exam Question Patterns

### Question Type: Multiple Choice

In PMP exam, most questions are **Multiple Choice Questions**:

#### Options:

- **4 choices** (A, B, C, D)
- **One correct answer** (in most cases)
- **Two clearly wrong answers**
- **One "close" answer** but not the best
- **One best answer** (Most Correct)

---

### Sample Question from Lecture:

#### Question:

**"In a Functional organizational structure, what is the role of the Project Manager?"**

**Options**:

**A)** Full authority over the project and team members

**B)** Limited authority and primarily acts as a coordinator or expeditor

**C)** Equal authority with the Functional Manager

**D)** No role in the project

---

### Analysis:

#### Option A: ‚ùå Wrong
- Full authority = **Projectized** (not Functional)

#### Option B: ‚úÖ Correct
- In Functional, PM = **coordinator only**
- **Very limited** authority

#### Option C: ‚ö†Ô∏è Close but wrong
- This describes **Balanced Matrix** (not Functional)
- In Functional: Functional Manager has **all** authority

#### Option D: ‚ùå Clearly wrong
- There's no "no role" - there's always some role

**Correct Answer**: **B**

---

## üß™ Solution Strategy

### Steps:

1. **Read the question carefully**
   - Did it specify structure type?
   - If not ‚Üí assume **Strong Matrix**

2. **Eliminate clearly wrong answers**
   - Usually 2 options are clearly wrong

3. **Compare the remaining two answers**
   - One is "close" but not the best
   - One is the **best** (Most Correct)

4. **Choose the best**
   - The answer that fits the context **most**

## üí° Additional Examples

### Example 1:

**Question**:
"You are a Project Manager in a large company. You need a software engineer for 3 weeks, but the IT department manager refused. What do you do?"

**Analysis**:
- Department manager refused ‚Üí PM has **limited authority**
- Structure: **Functional** or **Weak Matrix**
- **Cannot be** Strong Matrix (PM would have authority)

**Solution**:
- Negotiate with IT manager
- Request support from senior management
- Look for alternatives

---

### Example 2:

**Question**:
"You are a Project Manager. You decided to increase project budget by 20% and hire 5 new developers. What structure type?"

**Analysis**:
- Decided **independently** to increase budget and hire staff
- **Very high** authority
- Structure: **Projectized** or **Strong Matrix**

---

### Example 3:

**Question**:
"Mohammed works as an accountant in Finance department. He spends 3 days/week with Project A, and 2 days with Project B. Reports to Finance Manager and both PMs. What structure?"

**Analysis**:
- Works on **multiple projects**
- Reports to **multiple managers**
- Structure: **Balanced Matrix**
